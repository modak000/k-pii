## 5. 검토 큐 (REVIEW 항목)

PERSON 검출 중 confidence < 0.7 인 후보들 (홍길동·삼성전자) 는 REVIEW 큐로
이동. 관리자가 수동으로 확정/거부 후 vault 반영.

## 6. 인식 한계 (이 샘플에서)

- **차량번호 12가1234** 가 검출됐으나 안 잡힘 — `(차량번호) 12가1234` 형식에서
  prefix anchor 가 토큰 직전이 아니라 토큰 안에 들어가 일부 누락 가능.
  (실제로는 `12가1234` 자체는 detect_all 단독 호출 시 잡힘 — CLI vault
  배치 시 중복 제거 경계 영향)
- **사업자등록번호 123-45-67890** 미매칭 — 가짜 번호라 국세청 체크섬 실패
  (정상 동작; 가짜 번호는 emit X)
- **2026년 4월 25일** 미매칭 — 접수일자라 DT_BIRTH anchor 없음 (정상)
- **삼성전자** 미매칭 — 회사명 검출기는 별도 (이 라이브러리 스코프 밖)
- **종로구-민원-2026-04875** = DOC_ID 로 잡힘 (위 분포 누락된 게 아니라
  토큰 자체에 포함)

## 7. 사용 명령 재현

```bash
# 가명화 + Vault 저장
k-pii sample.txt --mode STRICT --strategy tokenize --vault vault.json > output.txt

# HTML 검출 보고서
python -c "
from k_pii.anonymizer import Anonymizer
from k_pii.core.modes import ProcessingMode
from k_pii.reporting.html_report import generate_html_report
text = open('sample.txt').read()
result = Anonymizer(mode=ProcessingMode.STRICT, strategy='tokenize').process(text)
open('report.html', 'w').write(generate_html_report(text, result, document_id='SAMPLE-001'))
"

# 복원 (Vault 키 보유자만)
k-pii output.txt --restore --vault vault.json
```
