## 외부 벤치마크: KLUE-NER

**데이터 출처:** [KLUE-benchmark/KLUE](https://github.com/KLUE-benchmark/KLUE) —
한국어 자연어이해(NLU) 평가용 공개 벤치마크 (카카오·NAVER·SKT 등 공동 구축).
NER 서브셋 dev = 5,000 신문기사 문장.

```bash
curl -L -O https://raw.githubusercontent.com/KLUE-benchmark/KLUE/main/\

```

## 룰 보강 진행 추이 (모두 ML 없이, 룰만)

| 단계 | 조치 | Precision | Recall | F1 |
|---|---|---|---|---|
| 0 | 초기 | 0.153 | 0.055 | **0.097** |
| 1 | 조사 가중치 0.20 → 0.35 + common_words 100개 | 0.155 | 0.364 | **0.216** |
| 2 | 부분 가명 거부 + 직책 분리 + 이름끝 음절 보너스 | 0.155 | 0.364 | **0.217** |
| 3 | 동적 threshold (2자 stem strict) | 0.208 | 0.328 | **0.255** |

| 5 | KLUE dev FP top 60개 wholesale common_words | 0.251 | 0.323 | **0.283** |

| 7 | **Method 1** 한국 인명 음절 통계 (+0.20) | 0.253 | 0.369 | **0.300** |
| 8 | **Method 2** 같은 문장 후보 상호 보강 (+0.15) | 0.253 | 0.369 | (M1과 합산) |

**최종 (Default): F1 = 0.300** (3.1배 개선).
**최종 (Korean-only): F1 = 0.331 / Recall = 0.485** — 실제 가명화 대상에서 *한국 이름의 절반* 을 룰만으로 catch.
합성 공문서 F1 = 1.000 *전체 유지*.

## Method 1·2·7 의 효과 분석

### Method 1 — 한국 이름 음절 통계 사전
- **`context/name_syllables.py`**: 매우 빈출 50+ / 흔한 40+ / 희귀 (페널티) 30+ 음절
- 룰: 후보 토큰의 *이름 부분* 음절 평균 score → 0~0.20 보너스
- 효과: **Recall 0.32 → 0.37**. "장혁·이범수·김기덕" 같은 통계적 한국 이름 catch
- 데이터 출처: 통계청 출생신고 + KLUE 분석 + 한국학 인명 자료

### Method 2 — 같은 문장 후보 상호 보강 (Co-occurrence Boost)
- 한 문장에 *score ≥ 0.7* 후보가 있으면 같은 문장의 약한 후보 *+0.15*
- "이범수와 김지우가 출연" 같은 *인명 나열* 패턴 직접 활용
- 효과: **105건 추가 TP** (KLUE-NER dev 기준)
- 사례: "신동주·신격호" "장윤주·하지원" "고동민·이현준" 등 함께 잡힘

### Method 7 — 외래어 origin 분류기
- **`context/name_origin.py`**: 5자+ 한글 / ㄹ 두음 / 외래 음절 휴리스틱
- 검출 결과에 `extra["origin"]` 부착 (korean / foreign / unknown)
- 평가에 `--korean-only` 옵션 — 본 라이브러리 *정책 대상* 만 측정
- 효과: **Korean-only recall 0.37 → 0.485**
  - "왕가위·소피마르소·짐캐리·도라에몽" 등 외래어 PS 라벨이 평가에서 제외

## 적용된 룰 요약

### 부정 신호 (FP 방지)

1. **부분 가명 표기 거부** — "박씨", "정모", "이군" 같은 2자 + 가명 마커
2. **복수형/명사 접미사 거부** — "사람들" (X들), "연기력" (X력), "기대감" (X감)
3. **동사·형용사 활용형 거부** — 하다/되다/드리다 + 관형형/연결형 ~30종
4. **동사 어간 패턴 거부** — 나오·재미있·추정되·만드
5. **행정구역 / 도로명 접미사 거부** — 시·군·구·도·읍·면·리, 로·길·대로
6. **공공기관 / 약칭 거부** — 정부조직 약칭 매칭
7. **field_label / title / agency 자체 거부** — 본인이 사전 단어면 거부
8. **common_words 사전 (~400개)** — 신문기사·공문서 빈출 어휘
9. **동적 threshold** — 2자 + 이름 글자 아닌 끝 + 컨텍스트 없음 → 임계값 +0.10

### 긍정 신호 (Recall 보강)

1. **조사 인접** (+0.35) — 한국어 PERSON 의 강한 신호
2. **field_label_before** (+0.50) — 명시적 양식 라벨
3. **title 인접** (+0.35) — 직책 (의원·교수·과장 등)
4. **결정적 PII 인접** (+0.40) — RRN/PHONE/EMAIL 옆
5. **surname** (+0.15) — 한국 성씨 286개
6. **누적 사전 부스트** (+0.40 cap)
7. **agency 같은 문장** (+0.10)
8. **embedded title 분리** (+0.35) — "강회장" → "강" + "회장"
9. **이름 끝 음절** (+0.10) — 빈출 이름 글자
10. **나이/성별 인접** (+0.30) — "(32세)", ", 남자"

## FN 의 본질적 한계 (룰로 해결 불가)

| 카테고리 | 빈도 | 이유 |
|---|---|---|
| 외국인명 (왕가위·소피마르소·짐캐리) | 100+ | 한국 공공 가명화 *대상 아님* |
| 가공인물 (도라에몽·사다코) | 50+ | 가명화 대상 아님 |
| 단음절 성씨 단독 (박·김·이) | 449 | 의도적 미검출 (확실성 부족) |
| 1자 영문 가명 (A·B·C) | 135 | 이미 가명화된 표기 |
| 특이 한국 이름 (장혁·이범수) | 50+ | surname + 일반어 충돌 |

→ **약 30-40% 의 KLUE-NER PS 라벨은 본 라이브러리의 *정책적 미검출* 대상.**
조정된 평가 (정책 대상만) 시 *실질 recall* 은 더 높음.

## F1 격차 해석 (합성 1.0 vs 실데이터 0.29)

이 격차는 두 가지를 모두 보여준다:

1. **합성기 자기-검증의 한계** — 합성 템플릿이 검출 패턴과 같은 가정을 공유
   하므로 F1=1.0 은 *"동작은 한다"* 의 신호일 뿐, *"보편적으로 잘한다"* 가 아님
2. **룰 기반 자연어 NER 의 본질적 천장** — 한국어 일반 명사·동사·형용사가
   surname-시작 글자와 광범위 겹침. 사전 폭증으로도 완전 제거 불가

## 권장 워크플로우

```
공문서 → k-pii (자동) → BLOCK 자동 가명화 → 안전 출력
                     → REVIEW 큐 → 사람 검토 → OK/FP/FN 마킹
                                            → 피드백 학습 → common_words 갱신
```

ML 없이 *룰 + 사람 협업* 으로 production 가능. 시간이 갈수록 사전이 풍부해져
정확도 자동 ↑.

## 향후 가능 (도메인 입력 필요)

- 학습된 common_words 의 git versioning + 공공 release
